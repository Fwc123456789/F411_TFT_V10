## TFT驱动层

### 1. 驱动架构 - ST7789控制器

**型号**: ST7789 LCD驱动IC
- **分辨率**: 320×240 pixels (RGB565, 16-bit)
- **接口**: SPI1 (PA5-SCK, PA7-MOSI)
- **控制引脚**:
  - RST (复位): RES_GPIO_Port
  - CS (片选): CS_GPIO_Port
  - DC (数据/命令): DC_GPIO_Port
  - BL (背光): BL_GPIO_Port

### 2. 通讯方案 - DMA + 轮询混合

```c
// 条件判断: 数据长度决定传输方式
void ST7789_SendMultiByte(uint8_t* dat, uint16_t len) {
    if (len > 64) {
        // 大数据块: 使用DMA传输 (异步)
        ST7789_SendMultiByte_DMA(dat, len);
    } else {
        // 小数据块: 轮询传输 (同步, 低延迟)
        HAL_SPI_Transmit(ST7789_SPI, dat, len, HAL_MAX_DELAY);
    }
}
```

### 3. 双缓冲DMA优化

**设计思想**: 解决清屏时的DMA冲突

```c
#define DMA_BUFFER_SIZE 1024

static uint8_t dma_buffer1[DMA_BUFFER_SIZE];
static uint8_t dma_buffer2[DMA_BUFFER_SIZE];
static volatile uint8_t active_buffer = 0;    // 当前活跃缓冲区
static volatile uint8_t dma_busy = 0;         // DMA忙标志

void ST7789_SendMultiByte_DMA(uint8_t* data, uint16_t len) {
    // 等待前一次DMA完成
    while (dma_busy) { ... }
    
    dma_busy = 1;
    
    if (len > DMA_BUFFER_SIZE) {
        // 分块传输大数据
        // 每次传输最多1024字节
    }
    
    // 复制数据到活跃缓冲区
    memcpy(active_buffer ? dma_buffer1 : dma_buffer2, data, len);
    
    // 启动DMA传输
    HAL_SPI_Transmit_DMA(...);
    
    // 切换缓冲区
    active_buffer = !active_buffer;
}

void ST7789_DMA_Complete_Callback(void) {
    dma_busy = 0;  // 解除DMA忙标志
}
```

### 4. 清屏性能优化

```c
void ST7789_Clear(uint16_t color) {
    // 分块清屏: 避免单次DMA传输过大
    #define ST7789_BUF_SIZE (240 * 2)  // 240像素宽 × 2字节/像素
    
    // 准备一行的颜色数据
    for (i = 0; i < ST7789_BUF_SIZE; i += 2) {
        ST7789_Buf[i] = color >> 8;      // 高字节
        ST7789_Buf[i + 1] = color & 0xFF; // 低字节
    }
    
    // 逐行传输: 共需240次DMA传输
    // 总数据量: 320×240×2 = 153.6KB
    for (i = 0; i < height; i++) {
        ST7789_SendMultiByte(ST7789_Buf, ST7789_BUF_SIZE);
    }
}
```

## 传感器驱动

### 1. DHT11 驱动 (湿度/温度)

**原理**: 单总线通信 (GPIO_B9)

#### 初始化序列
```c
uint8_t DHT11_Init(void) {
    DHT11_Rst();      // 复位
    return DHT11_Check();  // 检查应答
}

void DHT11_Rst(void) {
    DHT11_IO_OUT();        // 设置GPIO为输出
    DHT11_DQ_OUT_LOW;      // 拉低至少18ms
    HAL_Delay(20);
    DHT11_DQ_OUT_HIGH;     // 释放(变为浮空输入)
    delay_us(30);          // 等待20~40us
}
```

#### 通讯协议
```
主机发起信号:
主 ─────────┐
           │← 18ms ~  ┌──────────┐
高│        └──────────┘          └─────
  │
低│← 20~40us → 响应(低电平) → 主机读取
  
读数据:
bit=1: 低50us + 高26~28us  → 判断高电平时长
bit=0: 低50us + 高70us
```

#### 数据读取函数
```c
uint8_t DHT11_Read_Data(uint16_t *temp, uint16_t *humi) {
    DHT11_Rst();
    if (DHT11_Check() == 0) {  // 等待应答
        uint8_t buf[5];
        for (i = 0; i < 5; i++)
            buf[i] = DHT11_Read_Byte();  // 读取5字节
        
        // 校验: 前4字节和 = 第5字节
        if ((buf[0]+buf[1]+buf[2]+buf[3]) == buf[4]) {
            *humi = buf[0];  // 湿度整数部分
            *temp = buf[2];  // 温度整数部分
            return 0;  // 成功
        }
    }
    return 1;  // 失败
}

// 单字节读取 (8个bit)
uint8_t DHT11_Read_Byte(void) {
    uint8_t dat = 0;
    for (i = 0; i < 8; i++) {
        dat <<= 1;
        dat |= DHT11_Read_Bit();  // 逐bit读取
    }
    return dat;
}

// 单bit读取 (检测高电平时长)
uint8_t DHT11_Read_Bit(void) {
    // 等待低电平变高电平
    while (DHT11_DQ_IN && retry < 100) { delay_us(1); }
    while (!DHT11_DQ_IN && retry < 100) { delay_us(1); }
    
    delay_us(40);  // 等待40us
    
    if (DHT11_DQ_IN)
        return 1;  // 高电平 → bit=1
    else
        return 0;  // 低电平 → bit=0
}
```

#### 性能指标
- **响应时间**: ~2-5秒(传感器内部限制)
- **温度范围**: 0~50°C
- **湿度范围**: 20~90% RH
- **精度**: ±2°C, ±5% RH

### 2. DS18B20 驱动 (数字温度)

**原理**: 1-wire总线通信 (GPIO_B8)

#### 初始化
```c
void ds18b20_init() {
    onewire_init();  // 初始化1-wire总线
}
```

#### 温度读取流程
```c
float ds18b20_getTemp() {
    // ① 初始化总线 + 发送0xCC (跳过ROM匹配)
    onewire_init();
    onewire_sendbyte(0xCC);
    
    // ② 发送0x44 (开始温度转换)
    onewire_sendbyte(0x44);
    HAL_Delay(750);  // 等待转换完成
    
    // ③ 初始化总线 + 发送0xCC + 发送0xBE (读Scratchpad)
    onewire_init();
    onewire_sendbyte(0xCC);
    onewire_sendbyte(0xBE);
    
    // ④ 读取低字节和高字节
    uint8_t lsb = onewire_readbyte();
    uint8_t msb = onewire_readbyte();
    
    // ⑤ 数据转换
    uint16_t temp = (msb << 8) | lsb;
    
    // ⑥ 处理负温度 (补码)
    if ((temp & 0xF800) == 0xF800) {
        temp = (~temp) + 1;
        value = temp * (-0.0625);  // 负温度
    } else {
        value = temp * 0.0625;  // 正温度
    }
    
    return value;  // 单位: °C
}
```

#### 1-wire协议细节
```
初始化(reset):
主 ─────┐
       │← 480us ┌────────────┐
高│    └────────┘            └─ (从设备应答)
  │
低│← 500us → 从设备释放 → 等待500us

写bit (LSB first):
bit=0: 拉低60us + 释放1us
bit=1: 拉低1us  + 释放60us

读bit:
主 拉低1us + 释放: 从设备在10us内拉低则为0, 否则为1
```

#### 1-wire时序实现
```c

    uint32_t ticks = nus * 100;  // 100MHz
    
    // 计算tick差异, 实现微秒级延时
    while (tcnt < ticks) { ... }
}

void onewire_write(uint8_t dat) {
    if (dat == 0) {
        HAL_GPIO_WritePin(..., GPIO_PIN_RESET);
        HAL_Delay_us(60);      // 低60us
        HAL_GPIO_DeInit(...);  // 释放(浮空)
        HAL_Delay_us(1);
    } else {
        HAL_GPIO_WritePin(..., GPIO_PIN_RESET);
        HAL_Delay_us(1);       // 低1us
        HAL_GPIO_DeInit(...);  // 释放
        HAL_Delay_us(60);      // 高60us
    }
}

uint8_t onewire_read() {
    HAL_GPIO_WritePin(..., GPIO_PIN_RESET);
    HAL_Delay_us(1);           // 拉低1us
    HAL_GPIO_DeInit(...);      // 释放
    HAL_Delay_us(10);          // 等待10us读取
    
    uint8_t t = HAL_GPIO_ReadPin(...);  // 读取电平
    HAL_Delay_us(50);                    // 等待50us
    
    return t;
}
```

### 3. ADC 驱动 (模拟采集)

**硬件配置**:
- **三个通道**:
  1. `ADC_CHANNEL_1` (PA1): 外部模拟输入
  2. `ADC_CHANNEL_TEMPSENSOR`: 内部温度传感器
  3. `ADC_CHANNEL_VREFINT`: Vref内部参考电压

```c
void MX_ADC1_Init(void) {
    hadc1.Init.Resolution = ADC_RESOLUTION_12B;      // 12-bit分辨率
    hadc1.Init.ScanConvMode = ENABLE;                // 扫描模式
    hadc1.Init.NbrOfConversion = 3;                  // 3个通道
    hadc1.Init.SamplingTime = ADC_SAMPLETIME_480CYCLES;  // 480周期采样(低噪声)
    hadc1.Init.ClockPrescaler = ADC_CLOCK_SYNC_PCLK_DIV4;  // Prescaler=4
}
```

#### 采样流程
```
采样时间 = 480 + 12 = 492个ADC时钟周期
ADC时钟 = 100MHz / 4 = 25MHz
采样时间 = 492 / 25MHz ≈ 19.68μs (单通道)
3通道总耗时 ≈ 59μs
```

#### 数据读取方式
```c
// 因为DMA被TFT完全占用(SPI1_TX使用DMA2_Stream2)
// 采用轮询方式读取ADC (避免DMA冲突)

uint16_t adc_values[3];

void ReadADC(void) {
    // 逐通道读取
    HAL_ADC_Start(&hadc1);
    
    // 读通道1 (PA1)
    if (HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY) == HAL_OK) {
        adc_values[0] = HAL_ADC_GetValue(&hadc1);
    }
    
    // 读通道2 (温度)
    if (HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY) == HAL_OK) {
        adc_values[1] = HAL_ADC_GetValue(&hadc1);
    }
    
    // 读通道3 (Vrefint)
    if (HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY) == HAL_OK) {
        adc_values[2] = HAL_ADC_GetValue(&hadc1);
    }
    
    HAL_ADC_Stop(&hadc1);
}

// 电压计算
float channel1_voltage = adc_values[0] * 3.3 / 4095;  // PA1电压
float internal_temp = adc_values[1] * 3.3 / 4095;     // 温度传感器电压
float vrefint_voltage = adc_values[2] * 3.3 / 4095;   // Vrefint电压
```

#### 性能考虑
- **阻塞方式**: 等待前一通道转换完成
- **优化方案**: 可配置ADC+中断+DMA (但需不占用Stream2)
- **更新频率**: 主循环读取(间隔可控)

### 2. 长短按识别算法

**核心思想**: 基于TIM9计数器的持续时间判断

```c
void key_scan() {
    // 读取当前电平
    b1n = HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_12);  // KEY1当前状态
    
    if (b1n == 0 && b1o == 1) {
        // 下降沿: 按键按下 → 复位计数器
        TIM9->CNT = 0;
    } 
    else if (b1n == 0 && b1o == 0) {
        // 按键保持按下 → 检查是否为长按
        if (TIM9->CNT >= 10000) {
            // 长按(>10000 ticks ≈ 100ms @100MHz)
            // 此处可设置长按标志
        }
    } 
    else if (b1n == 1 && b1o == 0) {
        // 上升沿: 按键释放
        if (TIM9->CNT <= 10000) {
            // 短按(<100ms)
            KeyNum = 1;  // 设置按键值
        }
        // 长按释放: 无操作
    }
    
    b1o = b1n;  // 保存当前状态用于下次比较
}
```

**时序图**:
```
KEY_PIN高│     ┌─────────────┐         ┌─────┐
       │     │             │         │     │
       │─────┘             └─────────┘     └────
      
TIM9计数  0  →  增加  →  达到10000  或  释放时<10000
          ↑                ↓                  ↓
      按下时复位      长按判断           短按判断
```

### 4. 数据上传策略

**限制**: MQTT无法一次性发送多个消息(ESP8266固件限制)

**解决方案**: 使用标志位 + 主循环轮询

```c
// 全局标志位 (由中断/条件设置)
uint8_t sensor_read_pending;
uint8_t duqu, shangchuan1, shangchuan2, shangchuan3;

// 主循环中轮询执行上传
if (sensor_read_pending) {
    // 读取传感器
    DHT11_Read_Data(...);
    ds18b20_getTemp();
    sensor_read_pending = 0;
}

if (shangchuan1) {
    // 发送MQTT数据1
    printf("AT+MQTTPUB=0,\"8266_TX\",\"{...}\",1,0\r\n");
    HAL_Delay(500);  // 间隔
    shangchuan1 = 0;
}

// ... 其他数据上传
```

**为什么不在中断里直接printf**: 
- 中断中调用printf会导致延时, 违反中断最小化原则
- Printf涉及UART轮询, 影响中断响应时间
- 使用标志位解耦: 中断→置标志, 主循环→处理

## 性能优化

### 1. 显示优化

#### 问题: 全屏清屏耗时

**原因**: 153.6KB数据 (320×240×2) 逐行传输

```
清屏时间计算:
SPI1频率: 50MHz
每行数据: 240像素 × 2字节 = 480字节
传输时间/行: 480字节 × 8bit / 50MHz ≈ 77μs
总行数: 240行
总耗时: 240 × 77μs ≈ 18.5ms
```

#### 优化方案

**a) DMA优化**
- ✅ 已实现: DMA + 轮询混合(小数据用轮询, 大数据用DMA)
- ✅ 双缓冲区: 避免DMA冲突

**b) 局部刷新**
```c
// 只更新变化区域, 而非全屏
ST7789_FillRect(x1, y1, x2, y2, color);  // 局部矩形填充
// 比全屏快240倍!
```

**c) 分块传输**
```c
// 一次传输1024字节, 而非一次性153.6KB
for (i = 0; i < height; i++) {
    ST7789_SendMultiByte(line_data, 480);  // 逐行
}
```

#### 实际测试
- 全屏清屏: ~20ms
- 菜单切换: ~30ms (全屏+绘制UI)
- 数据更新: ~5-10ms (局部刷新)
- **帧率**: ~30-50 FPS (不含传感器读取)

## 关键技术实现

### 1. 图表滚动算法

**数据填满后的重绘策略**:

```c
if (data_index >= MAX_HISTORY) {
    // 数据已满100个点
    
    // ① 数组前移 (丢弃最老的数据)
    for (int i = 0; i < MAX_HISTORY - 1; i++) {
        history[i] = history[i+1];
    }
    data_index = MAX_HISTORY - 1;
    
    // ② 清屏并重绘整个图表
    DrawGraphGrid();
    
    // ③ 逐段绘制所有线段
    for (int i = 1; i < data_index; i++) {
        uint16_t x1 = 10 + ((i-1) * 220) / MAX_HISTORY;
        uint16_t x2 = 10 + (i * 220) / MAX_HISTORY;
        
        // DHT11温度曲线(红色)
        uint16_t y1 = 150 - (history[i-1].dht11_temperature * 110 / 50);
        uint16_t y2 = 150 - (history[i].dht11_temperature * 110 / 50);
        ST7789_DrawLine(x1, y1, x2, y2, RED);
        
        // 同理绘制湿度(蓝色)和DS18B20(绿色)
    }
}
```

**性能分析**:
- 数据前移: O(n) = 100次复制
- 图表重绘: O(n²) (每次移动都需要完整重绘)
- 优化方案: 使用环形缓冲避免数据移动

### 2. Bresenham直线算法

```c
void ST7789_DrawLine(uint16_t x1, uint16_t y1, uint16_t x2, uint16_t y2, uint16_t color) {
    // 直线光栅化: 用像素点连接两个坐标
    
    int dx = abs(x2 - x1);
    int dy = abs(y2 - y1);
    int sx = (x1 < x2) ? 1 : -1;
    int sy = (y1 < y2) ? 1 : -1;
    int err = dx - dy;
    
    int x = x1, y = y1;
    
    while (1) {
        ST7789_DrawPixel(x, y, color);
        
        if (x == x2 && y == y2) break;
        
        int e2 = 2 * err;
        if (e2 > -dy) { err -= dy; x += sx; }
        if (e2 < dx) { err += dx; y += sy; }
    }
}
```
