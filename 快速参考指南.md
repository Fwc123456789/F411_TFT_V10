## 📊 主循环执行流程

### main() 函数结构

```c
int main(void) {
    // ① 系统初始化
    HAL_Init();
    SystemClock_Config();
    
    // ② 外设初始化
    MX_GPIO_Init();
    MX_DMA_Init();
    MX_SPI1_Init();     // TFT
    MX_TIM9/10/11_Init();
    MX_ADC1_Init();
    MX_USART1_Init();   // ESP8266
    
    // ③ 应用初始化
    ST7789_Init();
    DHT11_Init();
    Key_Init();
    Send_AT_Commands_Simple();  // WiFi配置
    
    // ④ 启动定时器
    HAL_TIM_Base_Start(&htim9);
    HAL_TIM_Base_Start_IT(&htim11);
    
    // ⑤ 显示主菜单
    DrawMainMenu();
    
    // ⑥ 主循环
    while (1) {
        // A. 按键处理
        if (KeyNum) { ... }
        
        // B. 菜单绘制
        if (need_redraw) { DrawXXX(); }
        
        // C. 数据更新
        if (current_menu == MENU_TEMP_HUMI) {
            读取传感器 → 存储历史 → 更新图表
        }
    }
}
```

### while(1) 循环优先级

```
每个循环迭代 (~无固定周期, 取决于数据操作):

1. 检查 KeyNum (最高优先级)
   └─ KEY1/2/3/4 → 更新菜单状态

2. 检查 need_redraw 标志
   └─ 调用对应的 DrawXXX() 函数 (~30-50ms)

3. 如果在 MENU_TEMP_HUMI
   ├─ DHT11_Read_Data() (~2-5s一次, 阻塞~100ms)
   ├─ ds18b20_getTemp() (~5-10s一次, 阻塞~750ms) ← 关键瓶颈!
   ├─ UpdateGraph() (~30ms)
   ├─ DisplayCurrentValues() (~5-10ms)
   └─ data_index++ (环形缓冲)

4. 其他地方的轮询标志
   └─ sensor_read_pending, duqu, etc.

关键: ds18b20_getTemp() 的 750ms 延时
    → 建议改为非阻塞方案(参见技术深度分析文档)
```

## 💡 常见错误与解决

| 错误现象 | 可能原因 | 解决方案 |
|---------|--------|--------|
| 屏幕花屏/闪烁 | DMA冲突, SPI速率过高 | 降低SPI速率, 增加等待 |
| DHT11读取失败 | 时序崩溃, 中断延迟 | 禁用中断读取, 增加延时 |
| WiFi无法连接 | UART配置错误, 电源不稳 | 检查波特率, 增加延时 |
| 按键不响应 | GPIO未初始化, 引脚错误 | 检查Key_Init(), 验证引脚 |
| 内存溢出 | 历史数据过多, 堆栈溢出 | 减少MAX_HISTORY, 检查栈 |
| 菜单卡顿 | 阻塞操作过长 | 非阻塞化DS18B20读取 |

## 📞 快速诊断流程

### 新板测试检查清单

```
□ 1. 电源检查
   - 5V USB: 确保供电充足
   - 3.3V: 检查稳压器输出
   - GND: 确保接地良好

□ 2. 通讯检查
   - USART1: 使用串口助手测试 (115200)
   - printf测试: 发送任意字符查看回显

□ 3. TFT屏测试
   - 运行: ST7789_Init() + ST7789_Clear()
   - 观察: 屏幕是否全黑/全白

□ 4. 传感器检查
   - DHT11: 运行 DHT11_Init(), 检查printf输出
   - DS18B20: 运行 ds18b20_getTemp(), 查看温度值
   - ADC: 读取并显示ADC值

□ 5. 按键检查
   - 按下任意按键, 观察 KeyNum 变量

□ 6. WiFi检查
   - 运行 Send_AT_Commands_Simple()
   - 通过串口观察AT指令回复
```
